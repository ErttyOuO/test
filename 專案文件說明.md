# 📁 譯保通 InsureTech — 商業化訂閱頁面 專案文件說明

> **最後更新**：2026-03-01  
> **用途**：提供給協作者或 AI 快速了解本資料夾中每個檔案的功能、架構設計、關鍵常數與特殊約定。  
> **技術架構**：純前端 Demo（HTML + Vanilla JS），所有資料儲存在瀏覽器 `localStorage` / `IndexedDB`，無後端伺服器。

---

## 🏗️ 核心架構概念

| 概念 | 說明 |
|------|------|
| **localStorage 為資料庫** | 所有使用者帳號、金幣、簽到、貼圖、廣告活動皆存 localStorage（JSON 格式） |
| **auth.js 為核心引擎** | 所有頁面都 `<script src="auth.js">`，它負責帳號、資料讀寫、Header 注入 |
| **AD_SDK 為廣告引擎** | `ad-sdk.js` 以 IIFE 模式匯出 `AD_SDK` 全域物件，統管廣告投放/計費 |
| **IDB_FILES 為大檔儲存** | `idb-files.js` 將廣告素材（圖/影片）存入 IndexedDB，避免 localStorage 5MB 限制 |
| **角色系統** | `user` / `sponsor` / `agent` / `admin` 四種角色，透過 `requireRole()` 控制頁面存取 |

## 📂 檔案清單

### 🔧 共用 JavaScript 模組（所有頁面的基礎設施）

#### `auth.js` — 認證與資料同步核心
- **引用方式**：`<script src="auth.js">`（絕大多數頁面皆載入）
- **職責**：帳號註冊/登入/登出、使用者資料讀寫、統一 Header 注入、跨分頁同步、角色權限
- **關鍵全域函式**：

| 函式 | 說明 |
|------|------|
| `getUserData()` / `saveUserData(data)` | 讀寫當前使用者的 coins、VIP 等級等資料 |
| `getHistory()` / `addHistory(type, label, amount)` | 交易紀錄（上限 50 筆） |
| `isLoggedIn()` / `getAuthenticatedUser()` | 登入狀態檢查 |
| `getUserRole(userId)` / `getCurrentUserRole()` | 角色查詢 |
| `requireRole(...roles)` | 權限閘道（不符合時自動重導或顯示遮罩） |
| `getAiDailyLimit(data)` | 回傳 AI 每日上限（Free=15 / Basic=150 / Pro=1500） |
| `ensureDailyReset()` | 跨日自動重置 AI 使用次數 |
| `updateHeaderInfo()` | 刷新頁面頂部的 coins/VIP 徽章等顯示 |
| `getAvatar()` / `saveAvatar(base64)` | 頭像讀寫 |
| `getMyReferralCode()` / `applyReferralCode(code)` | 推薦碼系統 |
| `initAuthGate()` | 頁面進入點，DOMContentLoaded 自動執行 |

- **localStorage Keys**：

| Key | 說明 |
|-----|------|
| `localUsers` | 帳號清單陣列 `[{ id, displayName, username, password, role, ... }]` |
| `authUser` | 當前登入者 `{ id, displayName, username }` |
| `user_data_<userId>` | 個人資料 `{ is_vip, vip_level, coins, daily_ai_used, streak, ... }` |
| `coin_history_<userId>` | 交易紀錄陣列 |
| `avatar_<userId>` | 頭像 base64 |
| `referral_data_<userId>` | 推薦碼資料 |
| `sticker_unlocks_<userId>` | 已解鎖貼圖 ID 陣列 |

- **⚠️ 特殊約定**：
  - `local-admin` 的 `getUserRole()` 永遠回傳 `'admin'`（hardcode）
  - 新註冊帳號預設 `role: 'user'`
  - `requireRole()` 失敗時：sponsor/agent → 導向 `portal.html`；user → 顯示權限不足遮罩
  - 跨分頁同步靠 `storage` event + `setInterval` 1.5s 輪詢

---

#### `ad-sdk.js` — 廣告管理 SDK
- **引用方式**：`<script src="ad-sdk.js">`（需在 `auth.js` 之後載入）
- **匯出**：`AD_SDK` 全域物件（IIFE），另外提供 `showRewardedAd()` / `showBanner()` 全域快捷函式
- **職責**：Rewarded 影片廣告播放、Banner 渲染、廣告投放篩選、CPM/CPV/CPC 計費、事件記錄

- **核心常數**：

| 常數 | 值 | 說明 |
|------|-----|------|
| `REWARDED_DAILY_CAP` | `3` | 每日觀看上限 |
| `REWARDED_DURATION_FREE` | `15` 秒 | Free 用戶廣告時長 |
| `REWARDED_DURATION_VIP` | `5` 秒 | VIP 用戶廣告時長 |
| `REWARDED_REWARDS` | `{ wallet: {coins:3}, sticker: {coins:0}, checkin: {coins:0} }` | 各頁面獎勵 |
| `HOUSE_AD_CREATIVE` | HTML 平台宣傳 | 無任何廣告時的 fallback |
| `FALLBACK_RATES` | `{ banner: {impression:0.5,click:10}, rewarded: {impression:2,complete:5,click:10} }` | 預設費率 |

- **計費模型**：

| 廣告類型 | impression（曝光） | complete（完整觀看） | click（點擊CTA） |
|----------|:---:|:---:|:---:|
| Banner | 0.5 儲值金 | — | 10 儲值金 |
| Rewarded | 2 儲值金 | 5 儲值金 | 10 儲值金 |

- **Overlay 特殊功能**：
  - **關閉按鈕 (✕)**：廣告播放中點擊 → 確認彈窗 → 提前關閉不給獎勵、不扣額度
  - **👑 尊爵略過**：VIP 專屬金色按鈕，一鍵跳過並直接領取獎勵
  - **🚩 檢舉**：舉報不當廣告
  - 無廣告素材時使用 `HOUSE_AD_CREATIVE` 作為 fallback

- **localStorage Keys**：`ad_campaigns`、`ad_events`、`sponsor_profiles`、`sponsor_wallets`、`sponsor_wallet_history_<id>`、`fallback_creatives`、`system_settings`

- **⚠️ 特殊約定**：
  - Sponsor 餘額不足 → campaign 自動設為 `paused`
  - 投放優先排序：依 click rate 高者優先
  - VIP 判斷：`vip_level === 'basic' || 'pro'`

---

#### `idb-files.js` — IndexedDB 檔案儲存
- **引用方式**：`<script src="idb-files.js">`（需在 `ad-sdk.js` 之前載入）
- **匯出**：`IDB_FILES` 全域物件（IIFE）
- **職責**：將大型廣告素材（圖/影片）存入 IndexedDB，避免 localStorage 5MB 限制

| API | 回傳 | 說明 |
|-----|------|------|
| `IDB_FILES.save({ blob, type, name, sizeBytes })` | `Promise<fileId>` | 儲存檔案 |
| `IDB_FILES.get(fileId)` | `Promise<{ blob, url, type, name }>` | 讀取（含 ObjectURL） |
| `IDB_FILES.del(fileId)` | `Promise` | 刪除 |
| `IDB_FILES.clearAll()` | `Promise` | 清空全部 |

- **DB 名稱**：`AdManagerFiles`，store：`ad_files`，version：1
- **無外部依賴**

## ⚡ 給 AI 的快速參考

### 要修改使用者權限？
→ `auth.js` 的 `requireRole()` 和 `getUserRole()`，帳號角色存在 `localUsers[].role`

### 要修改 AI 諮詢額度？
→ `auth.js` 的 `getAiDailyLimit(data)` 函式

### 要修改廣告行為（時長/獎勵/上限）？
→ `ad-sdk.js` 最上方的常數區：`REWARDED_DAILY_CAP`、`REWARDED_DURATION_FREE`、`REWARDED_DURATION_VIP`、`REWARDED_REWARDS`

### 要修改計費費率？
→ `ad-sdk.js` 的 `FALLBACK_RATES`，或透過 admin-dashboard 設定存入 `system_settings`

### 要新增頁面？
→ 參考 `使用者權限.md` 中的步驟，同時在 `portal.html` 的 `ALL_PAGES` 陣列新增項目

### 要新增角色？
→ `auth.js` 的 `ensureDemoAccounts()` 加 Demo 帳號，`requireRole()` 加允許清單

### 腳本載入順序必須遵守？
→ **`auth.js` → `idb-files.js` → `ad-sdk.js`**（ad-sdk 依賴 auth.js 的全域函式）

### VIP 判斷邏輯？
→ `data.vip_level === 'basic' || data.vip_level === 'pro'`，不是看 `data.is_vip`（is_vip 只是衍生欄位）
