## 1. 專案整體定位

這個專案本質上是一個「保險 AI 平台前後端整合型專案」，包含：

- 前端多頁面網站：首頁、保單管理、社群、諮詢、DM、會員、廣告、後台。
- Node.js 主服務：提供網站靜態內容、MongoDB API、聯絡表單寄信、保單資料 CRUD。
- Flask AI 服務：處理一般 AI 問答，優先走 Vertex AI，失敗時退回 Gemini API Key。
- FastAPI 情緒/語音/推薦服務：
  - `backend/services/face_backend`：較完整的情緒、語音、保單推薦與 SSE 串流版本。
  - `backend/services/unified-chat`：整合聊天 API 的簡化版/原型版。
- 資料層：
  - `data/policy/policy_data.json` 提供平台保單資料庫。
  - MongoDB 儲存使用者、貼文、保單、投保建議。
  - 前端大量使用 `localStorage` 與 `IndexedDB` 存使用者狀態、留言、廣告素材、暫存檔案。

## 2. 系統主流程

### 2.1 首頁與一般頁面流程

- 入口檔是根目錄 [index.html](/C:/Users/luo/Desktop/IAFM/重新設計版面樣式/index.html)。
- 這個檔案會把路徑基準設到 `frontend/pages/`，所以首頁視覺與腳本實際來自：
  - [frontend/styles/style.css](/C:/Users/luo/Desktop/IAFM/重新設計版面樣式/frontend/styles/style.css)
  - [frontend/scripts/script.js](/C:/Users/luo/Desktop/IAFM/重新設計版面樣式/frontend/scripts/script.js)
  - [frontend/components/unified-chat-components](/C:/Users/luo/Desktop/IAFM/重新設計版面樣式/frontend/components/unified-chat-components)

### 2.2 AI 聊天流程

- 首頁載入後會延遲檢查 `http://localhost:8001/health`。
- 如果整合聊天後端有回應，就建立 `UnifiedChatManager`。
- `UnifiedChatManager` 再串接：
  - `EmotionDetector`：相機情緒分析。
  - `AIChat`：文字或語音送往聊天 API。
  - `AvatarSystem`：角色表情與動畫。
  - `VoiceModule`：錄音/送語音。
  - `TTSModule`：把 AI 回覆轉成語音。

### 2.3 保單資料流程

- 平台保單資料來自 [data/policy/policy_data.json](/C:/Users/luo/Desktop/IAFM/重新設計版面樣式/data/policy/policy_data.json)。
- 使用者自己的保單則由前端上傳，再交給 Node API 與 MongoDB 儲存。
- `face_backend` 會從 Node 的 `/api/policies/summary` 取使用者保單摘要，再混合平台保單資料，做推薦排序。

### 2.4 啟動流程

- 最完整的啟動入口是 [scripts/launchers/start_all.ps1](/C:/Users/luo/Desktop/IAFM/重新設計版面樣式/scripts/launchers/start_all.ps1)。
- 它會依序嘗試啟動：
  - Node server
  - face backend
  - unified backend
- 成功後預設開啟 `http://localhost:3000`。

## 14. 最後提醒

這個專案目前有三種狀態同時存在：

- 正式執行中的主線
- 仍可參考的舊方案
- 實驗/原型/模擬邏輯

所以實際改功能前，建議先確認三件事：

1. 這個頁面現在到底打哪個 API。
2. 這個功能是真資料、假資料，還是 localStorage 模擬。
3. 這份邏輯是在正式目錄，還是在 `archive/` 或原型檔裡。

只要先釐清這三件事，之後維護會快很多。
