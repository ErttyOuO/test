# 首頁輪播插圖比例修正草案 v2（針對你截圖這頁）

## 1. 本次聚焦
- 只處理首頁輪播「單張投影片」的視覺比例。
- 核心問題：插圖權重過高，文字被擠壓，閱讀主軸不夠穩。

目標是讓使用者先讀到文字，再被插圖輔助，而不是被插圖主導。

## 2. 為什麼會有「硬要放大」的感覺

對照目前樣式（已上線版本）：
- 插圖容器佔比偏大： [重新設計版面樣式/style.css#L283](重新設計版面樣式/style.css#L283)
- 插圖最大寬上限偏高： [重新設計版面樣式/style.css#L312](重新設計版面樣式/style.css#L312)
- 標題最大字級很大（最高到 3rem）： [重新設計版面樣式/style.css#L354](重新設計版面樣式/style.css#L354)
- 插圖發光層面積大且模糊強，會產生視覺膨脹： [重新設計版面樣式/style.css#L297](重新設計版面樣式/style.css#L297)

簡單說：
- 不是只有圖片尺寸大，而是「圖片佔比 + 發光範圍 + 字級節奏」一起讓畫面重心往右邊偏。

## 3. 修正原則（這次只做比例，不改風格）
- 文字區固定成主角（至少 58% 權重）。
- 插圖區退到輔助（30%~34% 權重）。
- 發光特效降噪，不關掉但縮小範圍與強度。
- 保留你現在的科技風配色與漸層，不做美術風格重設。

## 4. 建議參數（可直接落地）

### Desktop（>= 1280）
- `.ad-slide-content`: `flex: 0 0 58%`
- `.ad-slide-img`: `flex: 0 0 32%`
- `.ad-slide-img img`: `max-width: clamp(220px, 19vw, 300px)`
- `.ad-slide`: `gap: clamp(1rem, 2vw, 2rem)`
- 標題字級上限：`clamp(1.65rem, 2.5vw, 2.35rem)`

### Laptop / Tablet 橫向（1024 ~ 1279）
- `.ad-slide-content`: `60%`
- `.ad-slide-img`: `30%`
- 圖片上限：`max-width: 260px`
- 文字段落 `max-width: 100%`

### 平板直向 / 手機（<= 768）
- 維持上下排版（文字在上、插圖在下）
- 插圖高改為：`height: clamp(96px, 17vh, 140px)`
- 插圖寬改為：`width: min(64%, 220px)`
- 發光層透明度降到 `0.2~0.28`

## 5. 這頁（你截圖）專用微調

針對你貼的「升級會員方案」這張，建議加一個低侵入局部規則：
- 只針對第 5 張降低插圖存在感，不影響其他張
- 方式：
  - `.ad-slide:nth-child(5) .ad-slide-img { flex-basis: 30%; }`
  - `.ad-slide:nth-child(5) .ad-slide-img img { max-width: clamp(210px, 17vw, 280px); }`

這樣可立即改善「皇冠圖太搶眼、文字被壓」的問題。

## 6. 驗收標準（這次只驗比例）
- 第一眼焦點在標題，不是插圖。
- CTA 按鈕不被右側視覺壓迫。
- 三種寬度 1440 / 1200 / 1024 下，文字行長穩定、段落不擁擠。
- 手機 390 與 360 寬度下，插圖不覆蓋文字、版面不失衡。
