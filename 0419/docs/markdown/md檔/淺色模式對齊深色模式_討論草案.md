# 淺色模式對齊深色模式（討論草案）

## 0. 文件狀態
- 狀態：討論中，尚未執行程式修改
- 原則：先確認視覺目標，再落地實作

## 0.1 已確認需求
- 需求 1：淺色模式的頂端瀏覽列，樣式要參照深色模式，並維持同樣的分散排版節奏（導覽項目間距、群組對齊、左右留白一致）。
- 需求 2：淺色模式配色重新搭配，建立真正適合淺色場景的顏色系統，不是把深色配色直接提亮。
- 需求 3：首頁輪播主文案（AI 先判讀 -> 真人深度諮詢）重新設計，語句更精準且品牌一致。
- 需求 4：右上角「合作品牌推薦」先移除，現階段不顯示。
- 需求 5：淺色模式輪播比例要與深色模式完全相同，避免任何版型差異。

## 1. 目前已觀察差異
- 淺色模式在導覽列下方出現明顯灰白斷層。
- 深色模式的導覽列到首屏過渡是連續的，整體更完整。
- 目前淺色模式雖然輪播容器骨架已對齊，但「背景過渡層次」與「首屏貼合感」仍不一致。

## 2. 這輪討論目標
- 讓淺色模式的首屏結構與深色模式一致（容器寬度、圓角、間距、貼齊感）。
- 修掉導覽列下方的斷層感。
- 保留淺色模式應有的明亮感，不直接複製深色配色。
- 讓淺色模式頂端導覽列與深色模式使用同一套布局骨架（只換色、不換排版）。
- 建立淺色模式專屬色盤（背景、文字、邊框、按鈕、強調色）並套用到首頁關鍵區塊。
- 淺色模式與深色模式的輪播骨架參數一致（寬度、內距、圓角、高度、文字與插圖比例）。
- 輪播區塊資訊精簡：移除「合作品牌推薦」字樣，降低干擾。

## 3. 待你確認的決策點
### A. 導覽列下方過渡
- 選項 1：淺色模式加一層柔和漸層過渡（最接近深色視覺節奏）
- 選項 2：直接把首屏容器上推貼齊導覽列，取消灰白留白區
- 選項 3：維持留白但改為品牌色淡漸層（降低突兀）

### B. 首屏容器外框存在感
- 選項 1：保留邊框，但降低對比
- 選項 2：保留邊框，提升陰影柔和度
- 選項 3：邊框極淡化，靠陰影撐層次

### C. 淺色模式標題區可讀性
- 選項 1：維持現有字重，調深字色
- 選項 2：字重小幅增加（+100），字色微調
- 選項 3：僅調整副標字色，主標不動

### D. 淺色模式配色方向
- 選項 1：冷色科技白（藍灰底 + 青藍強調，最接近現有品牌）
- 選項 2：中性專業白（霧灰底 + 靛藍強調，商務感更強）
- 選項 3：柔亮高對比（暖白底 + 冷藍強調，可讀性最高）

### E. 輪播文案重設風格
- 選項 1：專業精準（短句、明確流程）
- 選項 2：親和導購（口語、降低門檻）
- 選項 3：科技感敘事（強調 AI 能力與效率）

## 4. 擬定修正方向（預設草案）
- 預設先採 A1 + B2 + C1 + D1 + E1。
- 先補導覽列下方柔和過渡，再微調輪播外框與文字可讀性。
- 不動輪播邏輯與內容文案，只改 CSS 視覺層。

### 4.2 已確認執行規則（需求 5）
- 輪播共用同一組骨架變數（淺色/深色）：
- `home-main-carousel-shell` 寬度與外距一致
- `home-ad-carousel` padding / min-height / border-radius 一致
- `.ad-slide` 的文字與插圖比例變數一致
- 只允許「配色與字色」主題化，不允許「結構數值」主題化

### 4.3 內容顯示規則（需求 4）
- 首頁輪播右上角「合作品牌推薦」先移除。
- 如後續要恢復，改由 feature flag 或註解保留可回復點。

### 4.4 首張文案重設草案（需求 3，E1）
- 主標：先由 AI 保險員快速判讀，再由真人顧問深度諮詢
- 副標：AI 先整理保障缺口與重點，真人再提供可執行的客製方案
- CTA：立即啟用 AI 判讀

- 區塊底色：#FFFFFF / #F6FBFF
- 主要文字：#10233D

## 6. 驗收標準
- 淺色模式導覽列到首屏看起來連續，不再有明顯斷層。
- 淺色與深色在版型節奏一致（不是顏色一致）。
- 首屏標題與副標可讀性達標，按鈕不被背景干擾。
- 桌機與手機都不出現新的留白或卡片跳動。
- 淺色模式在長時間瀏覽下不刺眼，且資訊層級（主標/副標/互動元件）可一眼辨識。
- 淺色/深色切換時，輪播尺寸與間距不跳動（視覺幾何完全一致）。
- 右上角不再顯示「合作品牌推薦」。
- 首張文案可在 3 秒內傳達「AI 先行、真人補強」流程。

## 7. 風險與回滾
- 風險：淺色過渡做太重，反而髒。
- 對策：先用低透明度漸層，必要時分兩版對比。
- 回滾：所有調整集中在首頁輪播樣式區，能快速回退。
