## 三、所有 API 與環境變數需求

### 服務 A — Node.js (`server/`)
| 項目 | 說明 |
|---|---|
| **MongoDB Atlas URI** | 已硬編碼在 `server/server.js` Line 20，可直接使用 |
| Node.js 版本 | >= 18 (使用 ES Module `import`) |
| npm 套件 | `express`, `cors`, `mongoose`, `bcryptjs`, `dotenv` |

**安裝指令：**
```bash

```

> ⚠️ 若遇到 `Cannot find module 'lib/express'`，請先刪除 `server/node_modules` 再重新 `npm install`。

## 五、啟動方式

### 方法一：一鍵啟動（推薦）

使用 `start_all.ps1`（PowerShell 腳本，支援中文路徑）：

```powershell
powershell -ExecutionPolicy Bypass -File start_all.ps1
```

或直接對 `start_all.ps1` 按右鍵 → 「以 PowerShell 執行」。

> ⚠️ **不要用 `.bat` 檔**：因為工具產生的檔案是 UTF-8 編碼，Windows CMD 無法正確處理，會出現亂碼。

啟動後會自動開啟：
- 🟢 PowerShell 視窗 1：Node.js (port 3000) → 自動帶起 Flask (port 5000)
- 🟢 PowerShell 視窗 2：FastAPI 面容情緒偵測 (port 8000)
- 🌐 瀏覽器 `http://localhost:3000`

### 方法二：手動啟動

| 順序 | 服務 | 指令 |
|---|---|---|
| 1 | Node.js (含自動喚起 Flask) | `cd server && node server.js` |
| 2 | FastAPI 面容情緒偵測 | `cd face_backend && uvicorn main:app --reload --port 8000` |

## 六、已知問題與解決方案

| 問題 | 原因 | 解法 |
|---|---|---|
| `Cannot find module 'lib/express'` | `node_modules` 損壞 | 刪除 `server/node_modules`，重新 `npm install` |
| `HTTP Error 403: Forbidden` (Vertex AI) | 未登入 gcloud | 已加入 Gemini API Key 自動降級，無需處理 |
| `upload_file` not found | `google-generativeai` 版本 0.3.1 無此 API | 已改用 `inline_data` (base64)，無需升級 |
| `.bat` 檔執行時亂碼或立即關閉 | UTF-8 BOM 編碼與 CMD 不相容 | 改用 `start_all.ps1` (PowerShell) |
| `面容情緒偵測` 路徑亂碼 | CMD/PS 傳遞中文路徑時編碼損壞 | 已複製到 `face_backend/` (純 ASCII 路徑) |
