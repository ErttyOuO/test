# 🎭 面容情緒偵測模組整理與整合指南

---

## 🎯 目的

本文件用於：

1. 避免誤刪「面容情緒偵測」中關鍵功能
2. 區分哪些檔案是核心、哪些是可移除
3. 為後續整合進 AI 視訊系統做準備

---

# 🧠 一、模組定位（非常重要）

「面容情緒偵測」不是單純舊專案，而是：

👉 AI 擬人視訊的核心來源之一

包含：
- webcam 操作
- 情緒偵測流程
- 前端互動邏輯
- 語音 / 文字輸入切換

⚠️ 結論：
**不能整包刪除，必須拆解保留**

---

# 🟢 二、【絕對保留】（不可刪）

## 🔹 前端核心（最重要）

路徑：
```
面容情緒偵測/frontend/
```

### 必留檔案：
- index.html
- app.js
- style.css
- config.js

### 原因：
這一組負責：
- 開啟攝影機（webcam）
- canvas 擷取畫面
- 情緒偵測 UI 流程
- 控制 AI avatar 狀態
- 處理語音 / 文字輸入

👉 未來會被整合進：
- consulting.html
- AI 視訊主系統

---

## 🔹 後端核心（AI邏輯）

路徑：
```
face_backend/
```

### 必留檔案：
- main.py
- gemini_client.py
- vision_client.py

### 功能：
- 情緒分析 API
- Gemini 對話
- 視覺辨識

👉 這是「AI大腦」

---

# 🟡 三、【建議保留但移出主線】

## 🔹 聲線測試模組

路徑：
```
面容情緒偵測/聲音聲線調整/
```

### 包含：
- index.html
- app.js
- style.css

### 用途：
- 測試不同 voice（溫柔女聲、沉穩男聲等）

### 建議：
移至：
```
_archive/voice-prototype/
```

---

## 🔹 重複後端（可疑）

路徑：
```
面容情緒偵測/.github/main.py
```

### 問題：
- 與 face_backend 重複
- 位置不合理（.github）

### 建議：
移至：
```
_archive/duplicate-backend/
```

---

## 🔹 文件

- AI 語音情緒助手.md

👉 可移至：
```
_archive/docs/
```

# 🔴 四、【可直接刪除】

## 🔹 Python 環境

- 面容情緒偵測/venv/

👉 理由：
- 非程式邏輯
- 可重建

## 🔹 快取檔

- __pycache__/
- *.pyc

👉 理由：

- 無需版本控制

# ⚠️ 五、重要風險提醒

## ❗ 1. frontend 不能刪

如果刪除：
```
面容情緒偵測/frontend/
```

會導致：
- webcam 無法使用
- 情緒偵測 UI 消失
- AI 視訊整合失敗

---

## ❗ 2. 目前尚未完全整合

目前狀態：

👉 consulting.html 仍可能透過「跳轉」使用該模組

表示：
- 尚未真正融合
- 還在依賴外部頁面

---

# 🚀 六、未來整合方式（重點）

## ❌ 錯誤方式

- 整個 frontend 複製貼上
- 用 iframe 嵌入

---

## ✅ 正確方式

從 frontend/app.js 中「拆功能」整合：

### 要搬的功能：

- webcam 啟動（getUserMedia）
- canvas 擷取
- 情緒偵測流程
- 語音輸入
- avatar 狀態控制

---

## 🎯 最終整合目標

👉 consulting.html 應包含：

- 視訊畫面
- AI Avatar
- 情緒反應
- 語音 + 打字
- 聊天紀錄
