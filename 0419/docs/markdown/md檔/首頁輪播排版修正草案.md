# 首頁輪播排版修正草案（重新設計版面樣式）

## 1. 範圍與目標
- 目標頁面：[重新設計版面樣式/index.html](重新設計版面樣式/index.html)
- 目標區塊：首頁主輪播（home-ad-carousel）
- 目標問題：排版穩定性不足、斷點視覺不一致、樣式權重衝突導致修了又被覆蓋

## 2. 目前問題診斷

### A. 樣式重複與覆蓋衝突（高風險）
- 輪播核心樣式在檔案前段定義一次，但在檔尾又有第二次定義，且含 !important，容易造成「調前面、後面又蓋掉」：
- 第一組輪播樣式起點：[重新設計版面樣式/style.css#L193](重新設計版面樣式/style.css#L193)
- 檔尾覆蓋樣式起點：[重新設計版面樣式/style.css#L5103](重新設計版面樣式/style.css#L5103)
- 導覽列也被檔尾全域放大覆蓋（會間接擠壓首頁第一屏）：[重新設計版面樣式/style.css#L5084](重新設計版面樣式/style.css#L5084)

### B. 手機版輪播高度與內容策略衝突（中高風險）
- 手機版視窗強制最小高度 420px，搭配卡片內容與絕對定位圖片，容易在小尺寸裝置出現遮擋或留白：
- 視窗高度設定：[重新設計版面樣式/style.css#L544](重新設計版面樣式/style.css#L544)
- 手機版圖片絕對定位與遮罩：[重新設計版面樣式/style.css#L568](重新設計版面樣式/style.css#L568)

### C. 輪播控制邏輯缺少互動保護（中風險）
- JS 有自動輪播與切換，但缺少 hover/focus 暫停、頁籤隱藏暫停、prefers-reduced-motion 兼容：
- 初始化邏輯位置：[重新設計版面樣式/script.js#L79](重新設計版面樣式/script.js#L79)

### D. 結構可維護性不足（中風險）
- HTML 結構合理，但 style.css 混入大量主題覆寫，輪播相關規則未明確模組化，後續調整容易牽一髮動全身：
- 輪播結構起點：[重新設計版面樣式/index.html#L78](重新設計版面樣式/index.html#L78)

## 3. 修正策略（先止血，再整理）

### Phase 1：排版止血
- 保留單一權威來源：輪播樣式只留一套，刪除檔尾重複覆蓋區段（含 !important 規則）。
- 手機版優先改為內容導向高度：
- 取消固定 min-height 420px，改用 clamp 與內容內距平衡。
- 圖片改為可退讓策略：小尺寸時降低透明圖層權重，避免壓字。

### Phase 2：互動穩定
- JS 增加以下行為：
- 滑鼠進入輪播或鍵盤 focus 到輪播時暫停自動播放。
- 視窗分頁隱藏時暫停，回到前景再續播。
- 若使用者偏好 reduced motion，預設關閉自動播放。

### Phase 3：結構整理
- 將輪播樣式集中成單一區塊並加註明確區間註解，降低未來回歸風險。
- 將輪播色彩/尺寸抽為區域變數，避免再用檔尾全域覆寫。

## 6. 驗收標準
- 桌機（1920/1440/1280）輪播卡片不裁切、文字不換行爆版、導覽按鈕可點。
- 平板（1024/820/768）輪播高度自然，圖片不壓文字，指示點位置穩定。
- 手機（430/390/375/360）無橫向捲軸、按鈕不重疊、CTA 可完整顯示。
- 輪播自動播放在互動時可暫停，切回分頁後行為正常。

## 7. 風險與回滾
- 風險：刪除檔尾覆蓋後，其他頁面若依賴同名全域樣式可能受影響。
- 對策：僅移除輪播與首頁主題相關覆蓋，非首頁元件先保留。
- 回滾：保留每次 patch 的最小差異，若視覺不符可分段回退（CSS 段落與 JS 段落分開）。
